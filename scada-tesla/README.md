# Tesla SCADA проект

Этот каталог содержит актуальный проект Tesla SCADA для системы «робот‑слесарь».  
Дамп состояния проекта на момент начала рефакторинга хранится отдельно в корне репозитория в каталоге `legacy/`.

## Структура каталога

- `project/`  
  Исходный проект Tesla SCADA для разработки в IDE.  
  Открывайте и редактируйте проект только из этой директории.

- `screens/`  
  Экспортированные экраны и скриншоты интерфейса Tesla SCADA.  
  Используются как визуальная документация и для отслеживания изменений HMI.

- `controls/`  
  Материалы по кастомным контролам: изображения, настройки, параметры, шаблоны.  
  Любые артефакты, связанные с пользовательскими элементами управления.

- `DB/`  
  Файлы баз данных проекта (журналы, архивы, конфигурационные таблицы и т.п.).

## Связь с каталогом `legacy/`

- `legacy/` в корне репозитория содержит контрольную точку всего стека (включая исходный проект SCADA) на момент старта работ по рефакторингу.  
- Для восстановления исходного SCADA‑проекта используйте файлы из `legacy/` и документы в `docs/`, а не изменяйте содержимое этого каталога.

## Работа с проектом

1. Открытие в Tesla SCADA  
   - Открывайте проект из `scada-tesla/project/` в соответствующей версии Tesla SCADA.

2. Внесение изменений  
   - При изменении экранов обновляйте экспорт/скриншоты в `screens/`.  
   - При добавлении или изменении кастомных контролов добавляйте/обновляйте материалы в `controls/`.  
   - Существенные изменения в структуре БД сопровождайте краткими заметками в `docs/` (например, `docs/architecture/`).
